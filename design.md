# Scrub Nurse — 游戏设计文档 (design.md)

## 一句话概述
第一人称器械护士模拟。术前把器械台整理到位,术中按医生需求把器械递到医生手中,用后取回并放回原位。写实风格,手术区以无菌布覆盖,无血无组织,焦点始终在器械与手。

## 灵感来源
玩家喜欢 Steam 上的 *Librarian: Tidy Up the Arcane Library!* 这类整理游戏。本项目不照抄,而是把"整理"玩法内核迁移到一个被低估的真实职业——手术室器械护士(scrub nurse)——上,并加入知识层(认器械、懂术式流程)。

## 为什么"器械护士"是优秀的整理题材
整理的六种爽感来源几乎全部命中,且有真实职业依据:

| 爽感来源 | 在本作中的体现 |
|---------|--------------|
| 归类归位 | 器械按用途归位到器械台指定区域 |
| 序列排序 | 器械按手术进程顺序摆放 |
| 配对成套 | 每台手术有固定器械包 |
| 多维约束 | 位置、顺序、原位归还同时满足 |
| 取舍清理 | 术中需持续把用过的器械归位 |
| 记忆追踪 | 记住每件器械的原槽位 |

额外的独特延伸:**递送**——整理是为了在医生伸手前把对的器械递过去,让"整理"有了应用目的,而非静态摆设。

## 核心循环

### ① 准备阶段(慢节奏 · 纯整理 · 教学)
- 推车里 6 件器械乱序堆放
- 画面侧边显示"摆放规范卡":6 个槽位 + 正确顺序
- 玩家点击器械拾取 → 点击槽位放入
- 位置/顺序正确变绿,错误变红
- 全部正确 → **自动 3-2-1 倒计时 → 进入术中**(无"开始手术"按钮)

### ② 术中阶段(快节奏 · 应用整理成果)
每步循环(共 6 步):
1. 医生需求弹出(图标 + 文字 + 语音),医生手伸出
2. 玩家点击台面对应器械 → 器械附在玩家手/视角上
3. 移动到医生手处 → **触碰即交付**(无需再点击)
4. **正确**:医生接住使用(手短暂收回)→ 使用后医生手再次伸出归还 → 玩家触碰取回 → **放回器械台原槽位**(提示"放回原位";放错则台面拒绝,弹回手中重放)→ 进入下一步
5. **错误**:医生手拒绝(推开/收回)→ 器械回到玩家手中 → 玩家自行放回原槽位 → 重新选择递送
6. 超时:扣分

### ③ 结算
6 步完成 → 星级 + 用时 + 正确率

## MVP 范围(明确边界)

**包含:**
- 准备阶段(归类 + 排序整理)
- 术中阶段(递送 + 取回 + 放回原位)
- 计分结算
- 拾起器械时显示名称与用途(知识注入)

**不包含(后续扩展):**
- 擦拭交互(取回直接放回原位)
- 数字快捷键递送
- 预判机制(提前备器械)
- 结尾清点(纱布/器械计数)
- 多术式 / 医生偏好
- 术前限时(准备阶段倒计时)
- 器械堆叠掉落物理

## 术式:虚构「门诊小手术」
虚构通用术式,不绑定具体疾病,降低资料准确度压力。流程线性 6 步,每件器械恰好用一次。

| 器械 (id) | 中文名 | 用途 | 需求时机 |
|-----------|--------|------|---------|
| scalpel | 手术刀 | 切皮 | 第 1 步 |
| hemostat | 止血钳 | 止血 | 第 2 步 |
| forceps | 组织镊 | 夹持组织 | 第 3 步 |
| scissors | 组织剪 | 分离切除 | 第 4 步 |
| needle_holder | 持针器 | 缝合 | 第 5 步 |
| gauze | 纱布 | 包扎 | 第 6 步 |

## 画面与表现
- 第一人称,玩家站在器械台前,低头看台面
- 对侧是手术区,**用无菌布巾完全覆盖**,只露出医生的手和伸出的需求
- 医生需求用**语音 + 图标 + 文字**提示
- 无血、无组织、无内脏
- 风格写实(参考 Librarian 的写实质感),场景为干净明亮的手术室
- 器械模型 MVP 阶段先用盒体 + 不同颜色/标签代用,后续替换真实风格模型

## 设计决策记录
- **去掉"开始手术"按钮**:准备完成自动倒计时进入术中,流程更顺
- **递送改为触碰交付**:器械附在玩家手上,触碰医生手即完成,有重量感而非"飞过去"
- **错误反馈用手而非表情**:只露手,故用"手拒绝/推开"表达不接受
- **跳过擦拭**:取回直接放回原位,聚焦整理动作
- **放回原位为软约束**:玩家可不放回,但 MVP 直接提示放回原位以降低复杂度;被拒绝的器械也需玩家自行放回
